About
What Is This?
A student-led competition challenging Maimonides students to build functional AI agents — systems that perform complex, multi-step tasks and deliver tangible value. Not wrappers. Not chatbots. Agents.
🎯

Build Something Real

Your agent should solve an actual problem for actual people. We judge the live, working product — not slides, not videos, not promises.

🔒

Blind Judging

All submissions are anonymized by a faculty advisor before reaching the judges. Names are only revealed after scores are locked.

🧠

Architect, Don't Outsource

We evaluate your ability to design logic, engineer prompts, and build interfaces. Using AI as a tool for creation — not a shortcut — is the standard.

📦

Submit & Lock

Submit a copy of your agent by the deadline. That copy is what gets judged. You can keep developing your own version afterward.

Judging
Official Rubric
Each submission is scored 0–4 across four categories, then multiplied by the category weight. Maximum score: 40 points.
0 — Complete failure 1 — Bare minimum 2 — Competent 3 — Strong 4 — Outstanding
Score Utility×4 (max 16) Creativity & Complexity×3 (max 12) Functionality×2 (max 8) User Experience×1 (max 4)
4 Extreme utility; solves a significant problem for a broad or valuable group; meaningfully addresses the problem's full scope Architecturally impressive; multi-component agent with substantial extension of LLM capability; genuinely brilliant design No functional issues found during testing; handles edge cases gracefully Intuitive immediately; clean, efficient interaction from first use
3 Clearly useful for a defined group or field; shows serious thought and practical relevance Very creative solution; advanced prompting or thoughtful, well-considered system design Minor issues only; core workflow operates end-to-end Clear after brief use; minor rough edges in flow or layout
2 Solves a legitimate but modest problem; limited in scope or depth Some original thought; rudimentary agent structure; limited external integration Concept works but notable features are incomplete or fragile Usable but requires some explanation or trial-and-error
1 Trivial or niche use-case; little improvement over existing tools Largely derivative; minimal extension beyond base LLM behavior Major intended functionality broken DQ Confusing or frustrating to navigate
0 No coherent or meaningful problem solved Copy-paste level; no meaningful agent logic Does not function DQ Barely usable; no effort toward usability
⚠️ A score of 0 or 1 in Functionality = automatic disqualification. Your agent must work.

📊 Score Formula

(Utility × 4)
+ (Creativity & Complexity × 3)
+ (Functionality × 2)
+ (User Experience × 1)

= Final Score (max 40)

⚖️ Tiebreaker Procedure

  1. Higher Functionality score wins
  2. If still tied → faster runtime wins
  3. If still tied → fewer required user inputs wins
  4. If still tied → judges deliberate via written justifications
