How to Build Your First Agent

A step-by-step walkthrough of building an AI agent from scratch using conversation with an AI. No prior coding experience needed — just an idea and persistence.

💡 What you'll build in this guide
We'll walk through building a Study Flashcard Generator — an agent that takes your class notes, extracts key concepts, and creates interactive flashcards you can quiz yourself with. By the end, you'll see a working demo and understand the process well enough to build your own agent for any problem.
1

Pick a Real Problem

Before you touch any AI tool, spend 10 minutes thinking about a problem you actually have. The best agents come from genuine frustration — something you've thought "I wish something could do this for me."

Ask yourself:
  • What repetitive task do I spend too long on?
  • What's something I avoid doing because it's tedious?
  • What tool do I wish existed for school, work, or daily life?
  • What multi-step process could be automated or made much faster?

For this guide, our problem is: "I take messy notes in class, and making study flashcards from them takes forever. I want to paste my notes and instantly get flashcards."

2

Describe It to the AI

Open your AI of choice (Claude, ChatGPT, Gemini, etc.) and describe what you want. Be specific about what the agent should do, not just what it is. Think about inputs, outputs, and the steps in between.

Conversation — Step 2
You I want to build a web app where I can paste my class notes into a text box, click a button, and get back a set of study flashcards. Each flashcard should have a question on the front and the answer on the back. I want to be able to click each card to reveal the answer. Can you help me build this? I want it to be a single HTML file I can open in my browser.
AI Great idea! I can definitely help you build that. Let me clarify a few things to make sure I build exactly what you want:
  • How many flashcards should it generate per set of notes? (e.g., 5, 10, as many as needed?)
  • What kind of questions — factual recall, conceptual understanding, or a mix?
  • Do you want to use an API key for the AI processing, or should I simulate it for the prototype?
  • Any specific subjects this is for, or should it be general-purpose?
You Make it generate 5-10 flashcards depending on how much content there is. Mix of factual and conceptual. Let's start with a simulated version so I can see the interface, then we'll add a real API later. It should work for any subject.
✅ Why this works
Notice how we gave the AI a clear input (pasted notes), a clear action (click a button), a clear output (flashcards with Q&A), and a clear interaction (click to reveal). The more specific your first message, the less back-and-forth you'll need.
3

The AI will give you a working first draft. It won't be perfect — that's fine. The goal is to get something running that you can see and interact with, then improve from there.

⚠️ A prototype is not a finished product
At this point you have a working interface with fake data. That's a great start! Many people stop here — don't. The difference between a competition entry that scores a 1 and one that scores a 3+ is what happens in the next three steps.
4

Iterate — Connect the Real AI

Now you ask the AI to replace the placeholder logic with an actual API call. This is where your agent starts doing real work. You'll need an API key from a provider like OpenAI, Anthropic, or Google.

Conversation — Step 4
You I have an OpenAI API key. Can you modify the generate() function to send my notes to the GPT API and get back real flashcards? I want the AI to return them as JSON so I can parse and display them.
AI Here's the updated generate() function. Replace your existing one with this. You'll need to add your API key where indicated: 
async function generate() {
  const notes = document.getElementById('notes').value;
  if (!notes.trim()) return alert('Please paste some notes first.');

  const output = document.getElementById('output');
  output.innerHTML = 'Generating flashcards...';

  const response = await fetch('https://api.openai.com/v1/chat/completions', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': 'Bearer YOUR_API_KEY_HERE'
    },
    body: JSON.stringify({
      model: 'gpt-4o-mini',
      messages: [{
        role: 'system',
        content: `You are a study assistant. Given student notes,
          generate 5-10 flashcards as a JSON array. Each object
          should have "q" (question) and "a" (answer) keys.
          Mix factual recall and conceptual understanding.
          Return ONLY the JSON array, no other text.`
      }, {
        role: 'user',
        content: notes
      }]
    })
  });

  const data = await response.json();
  const cards = JSON.parse(data.choices[0].message.content);

  output.innerHTML = cards.map(c =>
    `<div class="card" onclick="this.classList.toggle('show')">
      <strong>${c.q}</strong>
      <div class="answer">${c.a}</div>
    </div>`
  ).join('');
}
You It works! But sometimes the AI returns text before the JSON and it crashes. Also, when I paste long notes it takes a while and there's no indication anything is happening.
AI Good catches. Let me fix both:
  1. JSON parsing — I'll add a regex to extract just the JSON array even if the AI adds extra text
  2. Loading state — I'll disable the button and show a spinner while it's processing
Here's the improved version...
🔑 This is the most important step
Notice the pattern: try it → find what breaks → tell the AI what broke → get a fix. This loop is how 90% of agent development works. You don't need to know how to fix the code yourself. You need to be good at describing what went wrong.
5

Add Complexity — Make It an Agent

Right now you have a tool — it does one thing. To make it an agent, add multi-step logic, decisions, or capabilities that go beyond a single API call. This is where you stand out in the competition.

Conversation — Step 5
You This is good, but right now it's just a wrapper around one API call. I want to make it smarter. Can you add these features:
  1. Difficulty detection: After I answer all the cards, it should figure out which ones I got wrong and generate harder follow-up questions on those topics
  2. Subject detection: Auto-detect what subject the notes are from and adjust the question style (e.g., definition-heavy for biology, problem-solving for math)
  3. Export: Let me download my flashcards as a file I can import into Anki
AI Now we're building a real agent. Here's my plan:
  1. Self-assessment loop: After each card, I'll add "Got it" and "Missed it" buttons. When you finish a round, the agent will collect your misses, send them back to the API with a prompt asking for deeper questions on those specific topics, and generate a second round.
  2. Subject classification: Before generating cards, I'll add a first API call that classifies the subject and returns a strategy. The flashcard prompt will be dynamically adjusted based on this.
  3. Anki export: I'll format the cards as a tab-separated .txt file that Anki can import directly.
This means the agent now makes 2-3 chained API calls with logic in between — that's a genuine multi-step agent. Let me write the code...
🏗️ What makes it an "agent" vs. a "wrapper"
  • Wrapper: User input → one API call → display output. (Rubric score: 1-2 on Creativity)
  • Agent: User input → classify → decide strategy → generate → get feedback → adapt → generate again. Multiple steps, decisions, and loops. (Rubric score: 3-4 on Creativity)
6

Polish & Test

Before you submit, spend time on the details. Judges interact with your agent live — first impressions matter, and edge cases will be tested.

Conversation — Step 6
You The features all work. Now I want to make it look really polished. Can you:
  1. Make the design dark-themed and modern looking
  2. Add a smooth flip animation when revealing cards
  3. Handle errors gracefully — if the API fails, show a friendly message instead of crashing
  4. Add a character count so I know if my notes are too long
  5. Make it work well on mobile
AI Great instincts — all of these will directly improve your User Experience score on the rubric. Here's the complete polished version with all five improvements...
⚠️ Test these before submitting
  • What happens if you submit empty notes?
  • What happens if you paste a single word?
  • What happens if you paste 10 pages of text?
  • What happens if you paste something that isn't notes (e.g., song lyrics, random characters)?
  • What happens if your internet goes out mid-generation?
  • What happens if you click the button twice rapidly?
Every one of these that you handle cleanly is points on the Functionality rubric. Every one that crashes is points lost.

🎯 Summary: The Development Loop
  1. Idea — Pick a real problem you care about
  2. Describe — Tell the AI exactly what you want (inputs, outputs, interactions)
  3. Prototype — Get a working first version, even with fake data
  4. Connect — Wire up real AI capabilities (API calls)
  5. Extend — Add multi-step logic to make it a true agent
  6. Polish — UI, error handling, edge cases
  7. Test — Try to break it, then fix what breaks
  8. Submit — Lock your version by May 31 at 11:59 PM
🤔 But isn't the AI writing all the code for me?
You're the architect. The AI is the contractor. You decided what to build, how it should work, what features matter, how to handle errors, and what the user experience should feel like. The AI wrote code to your specifications — just like a professional software architect directs a team. That's exactly the skill this competition evaluates.
